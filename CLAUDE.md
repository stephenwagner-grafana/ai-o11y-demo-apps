## TL;DR — what this is

Two AI applications (NeonCart e-commerce + SupportBot internal helpdesk) deployed by one Helm chart to any Kubernetes cluster, instrumented end-to-end with OpenTelemetry, designed to populate a Grafana AI-observability dashboard with realistic data within ~10 minutes of install.

**The artifact is the apps.** Dashboards, the Sigil plugin, and the rest of the Grafana stack are consumed separately via the customer's Grafana Cloud. This repo only emits signals.

## Quick reference

### Install / verify / uninstall

```bash
./tools/install.sh    # interactive — writes .env, .helm-values-overrides.yaml, runs helm upgrade
./tools/verify.sh     # 9 checks; exits 0 when everything healthy
./tools/smoke-test.sh # exercises gateway, chatbot, mice trap, sb-router end-to-end
./tools/uninstall.sh  # helm uninstall + namespace cleanup
```

`install.sh` is the customer-facing contract. A new operator should clone, supply `.env`, run install, run verify, and get green within ~2 minutes (apps take ~10 min to produce dashboard-visible volume).

### Helm release + namespaces

- Release name: **`ai-o11y-demo-apps`**
- The release floats — `helm list -A` to find which namespace it landed in (often `claude-code` on the dev cluster, `default` on a clean deploy).
- Five app namespaces (chart-managed): `neoncart`, `support-bot`, `llm-gateway`, `ai-o11y-postgres`, `k6-loadgen`.

### Inner-loop iteration on a running cluster

```bash
# Reapply chart only (e.g., values change)
helm upgrade ai-o11y-demo-apps ./helm -f .helm-values-overrides.yaml -n <release-ns>

# Pick up new images (pullPolicy=Always, CI publishes :latest on main push)
kubectl rollout restart deploy -n <ns> <deploy>

# Full clean cycle
./tools/uninstall.sh && ./tools/install.sh && ./tools/verify.sh
```

### Secret / config files (all gitignored)

- `.env` — secrets the user supplies once; `install.sh` reads from it
- `.helm-values-overrides.yaml` — generated by `install.sh` from `.env`
- `helm/config/users.yaml` — generated via `tools/regenerate-users.py --seed N`

## Architecture in 6 lines

- **k6 loadgen** (one pod, central) drives both apps' websites with realistic traffic. Hand-written K6, not generated.
- **NeonCart web** (public e-commerce) → `nc-chatbot` + `nc-gift-finder`. AI is *optional* (drives an adoption % story).
- **SupportBot web** (internal "Ask Acme") → `sb-router` (LLM classifier) → `sb-billing` / `sb-tech-support` / `sb-account-management`. AI is *mandatory*.
- All specialists call **llm-gateway**, which holds pricing, computes cost, enforces per-provider caps, exposes `/open`.
- All pods emit OTLP **direct to Grafana Cloud** (no bundled collector). If the customer has Alloy, they override the endpoint env var.
- Single **Postgres** (NC catalog/cart/orders + SB pgvector knowledge base, separate schemas).

## Locked decisions (do not re-litigate)

- **Exactly 2 NC specialists** (`nc-chatbot` + `nc-gift-finder`) and **3 SB domain specialists** + router. Adding a 4th SB specialist is fine; cutting below 2 makes the router theater.
- **Anthropic is the only mandatory provider.** OpenAI / Gemini / Ollama are optional. Gateway routes randomly across *configured* providers, not all four.
- **Default model is Claude Haiku.** Sonnet/Opus only when latency/quality contrast needs to surface in panels.
- **Cost is computed in the gateway**, not pulled from Sigil. Sigil computes cost server-side but exposes it only in its Analytics UI — no Prom metric (verified 2026-05-22). Gateway carries `pricing.yaml` mirroring Sigil's canonical rates and emits `gen_ai.client.cost.usd`. When Sigil ships a native Prom cost metric, delete `pricing.yaml` — one PR.
- **Two-tier routing by `X-Caller-Type` header.** `synthetic` (loadgen) is gated by provider caps and `/open`. `interactive` (humans hitting the websites) always goes to Claude, ungated. Don't gate humans.
- **Provider caps are admission control, not sampling.** Gateway routes or 429s; never probabilistically drops. No business-hours clock-gating.
- **Claude is the throughput governor.** When Claude budget closes, loadgen watches `providers.anthropic` (not `any_open`) and stops spawning new AI VUs. NC's non-AI cohort keeps shopping; SB users visibly disappear from the dashboard. One knob (Claude $/day) sizes the whole demo.
- **OTLP direct to Grafana Cloud.** Standard env vars (`OTEL_EXPORTER_OTLP_ENDPOINT` + `OTEL_EXPORTER_OTLP_HEADERS`). No bundled Alloy.
- **Sigil Python provider wrappers** are used in the gateway for Anthropic / OpenAI / Gemini. Ollama is hand-rolled (`client.start_generation` + raw HTTP) — no wrapper exists.
- **OTel Tracer + Meter providers must be configured BEFORE Sigil client creation.** Otherwise telemetry goes to no-op and is silently lost.
- **Apps are model/provider agnostic.** Specialists ask the gateway for "an LLM." Dashboards reveal which one ran. Never bake model names into specialist code.
- **Sticky provider routing per-conversation.** A conversation stays on one model for coherent cost/latency analysis.

## By-design oddities — these are NOT bugs

- **`"show me mice"` returns a 500.** `nc-chatbot`'s `search_products` tool hardcodes a query against a column (`species`) that doesn't exist. The cascading Postgres error through browser → nc-web → nc-chatbot → postgres is the signature demo trace. Keep it.
- **AI Adoption % can exceed 100%.** `neoncart_session_used_ai_total` increments per AI call, not once per session. Cosmetic — a dedupe would fix it but it's not a priority.
- **25% AI adoption is unrealistic** for real e-commerce (single-digit typical). Demo legibility wins. Disclose in customer conversations, don't tune down.
- **Random provider routing is a demo choice, not a production pattern.** Real prod uses one provider + fallback. Disclose, don't redesign.
- **Hard $20/day Claude cap** is the original deploy's personal budget. Customer deploys should set their own via `global.caps.anthropic.usdPerDay`.

## Scope cuts — deliberately NOT here

If you find yourself rebuilding any of these, stop and confirm with the user:

- Tiered $-threshold sampling (the ObserVIBElity $40/$60/$80 pattern). Useless on a clean deploy.
- Dynamic Ollama model pool with sticky-per-session + 100-req lifecycle.
- Business-hours gate for Claude. Caps replace it.
- A UseCase ABC / registry / generated K6 scenarios. K6 here is hand-written in `loadgen/`.
- Per-evaluator dashboards / auto-registration on install. Evaluators are paste-from-`docs/EVALS.md` or one-shot via `tools/create-evaluators.py`.
- Public DNS / TLS automation. Optional Ingress objects ship via chart values; customer wires the rest.
- Scripted failure narratives (data theft, retrieval drift, email cascade). Base = steady ambient AI traffic; those belong in a separate `extensions/` location or sibling repo.

## Known issues (flagged, not priorities)

- **No log↔trace correlation.** Logs reach Loki but no `trace_id` field attaches despite `OTEL_PYTHON_LOG_CORRELATION=true`. Likely `LoggingInstrumentor` interaction with existing `logging.basicConfig()` calls.
- **No public DNS for `supportbot.wombatwags.com`.** Internal traefik works; public Cloudflare record missing.
- **K6 → web traceparent injection not implemented.** Loadgen requests don't share trace_id with downstream service spans.
- **Evaluator metrics empty until evaluators are registered.** Run `tools/create-evaluators.py` against the customer's Sigil.

## Conventions

- **Python**: Ruff lints in CI (`lint.yml`). No `ruff.toml` yet, so the job runs soft-fail — but write code that would pass strict ruff. Python 3.12.
- **Images**: GHCR at `ghcr.io/stephenwagner-grafana/ai-o11y-demo-apps/<component>:latest`. CI (`build.yml`) builds per-component on every push to `main`. `pullPolicy: Always` in the chart picks them up — restart deployments to roll.
- **Branching**: `main` only. PRs welcome but the repo is single-maintainer; direct commits to main are fine.
- **Secrets**: never commit. `install.sh` writes `.env` and `.helm-values-overrides.yaml`, both gitignored.
- **Generators overwrite hand-edits.** `tools/regenerate-users.py` and `tools/regenerate-products.py` rewrite their target YAML. Hand-edits survive across restarts only if the generator isn't re-run.
